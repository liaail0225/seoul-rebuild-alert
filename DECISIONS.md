# 결정 기록 (Architecture Decision Log)

모든 중요한 결정의 이유·대안·날짜를 기록한다. 새 결정은 아래에 추가.

## 2026-07-18

### D1. 런타임 = Node.js 24
- **이유**: 사용자 PC에 Node만 설치되어 있음(Python 없음) → 로컬 테스트 가능해야 함.
- **대안**: Python — 로컬 실행 불가로 기각.

### D2. 실행 = GitHub Actions cron, 저장 = Supabase, 알림/UI = 텔레그램
- **이유**: 사용자 선택(무료 클라우드). PC가 꺼져 있어도 매일 동작. Supabase 계정 이미 보유.
- **대안**: 로컬 PC 스케줄러(PC 꺼짐 문제), 유료 서버(관리 부담) — 기각.
- **주의**: repo는 public이어야 Actions 무료 무제한. 코드에 비밀값·개인 메모 절대 금지.

### D3. 기준 데이터(source of truth) = 정비몽땅 사업장 목록 스크래핑
- **이유**: 서울 열린데이터광장 OA-2253(정비사업 현황 API)이 2026-07-18부로 서비스 종료됨을 실호출로 확인.
  정비몽땅 목록은 서버 렌더링 HTML로 1,134개 사업장 + 14단계 진행단계 제공, pageSize=200 동작 확인.
- **대안**: OA-22856(분기별 엑셀) — 갱신 주기가 느려 보조/교차검증용.
- **원칙**: 추진단계는 이 소스에서만 갱신. AI·뉴스는 단계를 절대 바꾸지 못함.

### D4. 위원회 심의결과 = 네이버 뉴스(보도자료 커버) + 정비몽땅 고시/공고
- **이유**: 도시계획위원회 정보 API(OA-20297)도 같은 날 종료. opengov 위원회 회의정보는 2024년 이후 갱신 지연 확인.
  심의 통과·상정은 언론 보도가 가장 빠르고 확실하게 커버.
- **대안**: 서울시 보도자료 게시판 직접 스크래핑 — SPA 구조로 복잡, v2 후보.

### D5. 첨부파일 다운로드 = EUC-KR 인코딩 POST
- **이유**: 정비몽땅 첨부(dwldWithCsrf.do)는 한글 파일명을 UTF-8로 보내면 실패.
  dwldFileNm을 EUC-KR percent-encoding으로 POST해야 실제 파일 수신 (실측 검증).
- 판독: PDF(pdf-parse), HWPX(zip+XML). 구형 HWP는 v1에서 "직접 확인 필요"로 표시.

### D6. 알림 = 하루 1회 09:00 KST 통합 다이제스트 (즉시 알림 없음)
- **이유**: 사용자 지시. 고시·단계 변화는 분 단위 속보성이 없음.
- 우선순위 규칙(alertRules.js)은 AI가 아닌 코드가 결정: 초기 단계(추진위·조합설립·구역지정) 진입과
  설립 직전 신호(창립총회·인가 신청·심의)가 🚨 최우선 섹션.

### D7. 봇 명령 = 15분 폴링 (웹훅 없음)
- **이유**: GitHub Actions만으로 운영(배포 인프라 0개). 명령은 조회용이라 지연 허용.
- **대안**: Supabase Edge Function 웹훅(즉답) — 복잡도 증가로 v1 제외, v2 후보.

### D8. AI = OpenAI gpt-5.4-mini/nano + 월 $5 하드 캡
- **이유**: 사용자 보유 키. costGuard가 호출 전 예산 확인, 기능별 비용을 ai_analyses에 기록.
  예산 초과 시 AI만 중단되고 규칙 기반 알림은 계속 동작.
- AI의 권한: ai_analyses 테이블에 제안을 쓰는 것뿐. 단계 판정·알림 여부·데이터 삭제는 코드 규칙만.

### D9. 이주와 철거는 별도 단계
- **이유**: 사용자 지시 — 기간이 다름. stages.js 표준 체계에 반영.

### D10. 관심단지 기준 = DB watchlist (엑셀은 최초 시드)
- **이유**: 기준 데이터는 한 곳(원칙). 이후 추가/삭제는 텔레그램 봇 명령으로만.

### D11. data.go.kr 실거래가 API — 호출 방법 실측
- 사용자가 활용신청 완료(2026-07-18): 아파트 매매 실거래가(상세 포함), 분양권전매, 건축HUB 인허가.
- **호출 시 serviceKey는 인코딩 없이 그대로, User-Agent는 브라우저 UA 필수** (기본 curl UA는 WAF 차단).
- v1에서는 미사용. v2 후보: /단지 명령에 최근 실거래가 표시, 사업성 참고 지표.
  (사업장→법정동코드→단지명 매핑이 필요해 별도 작업으로 분리)
